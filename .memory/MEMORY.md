# Repository Memory

## Stable Context
- **需求說明必須完整**  
  每一個 Issue 在建立時應同時提供：  
  1. 問題或功能的具體描述  
  2. 預期的交付成果與驗收標準  
  3. 需求的優先級或時程（若已知）  
  4. 類別標記（Bug / Feature / Improvement 等）  

- **Agent 工作流程**  
  1. 每日自動抓取最近 30 天、上限 100 筆的 Issue。  
  2. 依據 Issue 內容產出「Agent Activity」報告，並辨識跨 Issue 主題。  
  3. 若發現資訊缺口，立即在 **Open Loops** 中列出待主人補充的項目。  
  4. 只在 **Shared Manual Notes** 中手動維護長期規則、決策與限制，系統不會自行覆寫此檔。  

- **資料來源原則**  
  - 只有 GitHub Issue / Comment 被視為原始資料。  
  - 手動筆記（`shared/manual.md`）僅作為摘要與規則存放，不會直接複製 Issue 原文。  

- **語言與格式**  
  - 所有記錄與說明必須使用繁體中文（zh‑TW）。  
  - Markdown 為唯一允許的輸出格式，以保持可讀性與可程式化處理。
