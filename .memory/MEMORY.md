# Repository Memory

## Stable Context
- **Repository**: `MSIT12806/WillClaw`  
- **Issue Management**  
  - 所有 Issue 必須在建立時提供 **完整需求說明**（背景、目標、驗收條件）。  
  - 建議使用統一的 Issue 模板，以避免「資訊缺口」的情況。  
  - 每個 Issue 應至少指派一位負責人並加上適當的標籤（如 `needs‑info`、`in‑progress`），以提升可見性與追蹤效率。  
- **資訊同步**  
  - 任何來自 Telegram、Slack 或其他即時通訊的討論必須同步至 Issue 討論串，避免資訊斷層。  
- **Agent 行為原則**  
  - 龍蝦群會每日掃描所有開放的 Issue，蒐集活動摘要，並在資訊不足時標記為 **Open Loop**。  
  - 只讀取 **GitHub Issue / comment** 作為原始資料來源，**不會**覆寫手動維護的 `shared/manual.md`。  
- **工作範圍**  
  - 只關注最近 30 天內、狀態為 `open` 的 Issue（上限 100 件）。
