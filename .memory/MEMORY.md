# Repository Memory  

## Stable Context  
- **Repository**: `MSIT12806/WillClaw` 為本龍蝦小隊服務的主要代碼庫。  
- **Issue 工作流程**  
  - 所有工作皆以 GitHub Issue 為入口，必須在 Issue 內提供 **完整需求說明**（背景、目標、驗收條件）。  
  - 缺少說明的 Issue 會被標記為 *資訊不明*，龍蝦們會暫停任何實作，直至主人補齊。  
  - 建議使用 **Issue 模板** 以確保每次建立都具備上述欄位。  
- **溝通渠道**  
  - 主要即時訊息來源為 Telegram / Slack，若有相關討論必須在 Issue 中附上連結或摘要，避免資訊斷層。  
- **標籤與指派**  
  - 為提升可見性與追蹤效率，所有 Issue 應在建立時即加入適當 **Label**（如 `needs‑clarification`、`in‑progress`）並指派負責人。  
- **決策紀錄**  
  - 任何決策皆須在 Issue 討論串或專門的決策文件中留下文字紀錄，龍蝦們不會自行假設方向。  

## Recent Themes

- **外部訊息未同步**：多次提到可能存在的 Telegram/Slack 討論，但未被納入 Issue，造成資訊斷層。  
- **缺乏決策**：截至 2026‑03‑31，尚未有任何正式決策產生，所有行動都在等待需求補齊。
