# Repository Memory  

## Stable Context  
- **Issue‑driven工作流程**：本倉庫以 GitHub Issue 為主要任務單位，所有工作皆應透過 Issue 進行說明、指派與追蹤。  
- **記憶層級**  
  - `shared/manual.md` 為人工維護的長期記憶，僅保存**穩定規則、長期決策、常見限制與 repo 習慣**，不會被自動覆寫。  
  - `.memory/agents/…` 為自動產生的臨時記錄，用於追蹤每日快照與 open loop。  
- **協作慣例**  
  - 每日快照由「龍蝦」 agents 產出，蒐集當天所有 active issue 的狀態與變化。  
  - 任何 **完整需求說明、指派、標籤** 必須由主人（Owner）提供，agents 只負責蒐集與提醒。  
- **資訊完整性原則**：Issue 必須至少包含 **標題 + 需求描述**，缺少任一項即視為「資訊缺口」， agents 會將其標記為 open loop。
