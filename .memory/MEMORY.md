# Repository Memory

## Stable Context
- **Issue 管理慣例**  
  - 每個 Issue 必須至少包含 **標題**、**描述**（說明背景、目標與預期成果）以及 **負責人**（若已知）。  
  - 為了讓龍蝦團隊能即時展開工作，建議在建立 Issue 時同步加入 **標籤**（如 `feature`、`bug`、`enhancement`）與 **時間/資源估算**。  
  - 任何決策（功能範圍、優先順序、技術選型）應以 **comment** 形式留下，方便後續自動摘要。  

- **每日記憶快照流程**  
  - 龍蝦每天會掃描過去 30 天內的所有 open Issue（上限 100），產出 **Agent Activity**、**Cross‑Issue Themes**、**Decisions** 與 **Open Loops**。  
  - 產出結果僅作為 **curated** 記錄，絕不直接複製原始 Issue 內容。  

- **共享手動筆記 (`shared/manual.md`)**  
  - 用於保存 **長期穩定規則、決策與常見限制**，不會被自動流程覆寫。  
  - 任何新規則或決策若需永久保存，應先寫入此手動筆記，再由龍蝦在日常快照中引用。  

- **語言與風格**  
  - 所有記錄皆以繁體中文撰寫，語氣保持「龍蝦在替主人記憶」的口吻，同時保持工程可讀性。
