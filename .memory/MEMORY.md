# Repository Memory

## Stable Context
- **工作流程核心**  
  - 所有任務皆以 GitHub Issue 形式登錄。  
  - 每個 Issue 必須包含 **明確的目標說明、需求描述**，並在必要時附上外部討論（如 Telegram、Slack）連結或摘要。  
  - Issue 建立後應立即 **指派負責人** 並 **加上適當標籤**（例如 `needs‑info`、`in‑progress`、`blocked`），以利追蹤與排程。  

- **Agent 角色**  
  - 龍蝦團隊每日掃描 repo 中的 Issue，彙整狀態、發現的共通主題與未完成項目，並以 **compact‑memory** 方式寫入長期記憶檔案。  
  - 任何 **決策** 必須由主人提供，Agent 只負責蒐集、整理與提醒。  

- **共通限制**（從手動筆記與日誌觀察得出）  
  - 只能依賴 **GitHub Issue / comment** 作為原始資料來源；其他平台的訊息必須先被手動匯入或以連結形式引用。  
  - 手動維護的 `shared/manual.md` 只作為 **長期規則與決策的存放處**，不會被自動覆寫。
