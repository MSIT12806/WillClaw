# Repository Memory  

## Stable Context  
- **Repository**: `MSIT12806/WillClaw`  
- **Agent identity**: 一群「龍蝦」(agents) 以爬行、鉗子、抓取的隱喻，負責持續監控 Issue、整理資訊、標記未完成項目，並將結果寫入 `.memory`。  
- **工作流程**  
  1. **監測 Issue 列表**：每日掃描所有開放的 Issue。  
  2. **資訊抽取**：若 Issue 只有標題或空白內容，標記為「資訊缺口」並加入 **Open Loops**。  
  3. **記錄與蒸餾**：將每日快照的重複訊息濃縮成長期可重用的上下文，避免逐段複製原始日誌。  
  4. **共享手動筆記**：`shared/manual.md` 保存 **穩定規則、長期決策、常見限制、repo 習慣**，Agents 只讀取不覆寫。  
- **長期規則（從手動筆記推測）**  
  - 只在 `shared/manual.md` 中保留 **穩定規則**、**長期決策**、**常見限制**、**repo 習慣**。  
  - 原始 Issue / comment 為唯一事實來源，Agents 不直接複製其全文。  
  - `MEMORY.md` 必須是 **curated** 的長期記憶，非原始日誌或索引頁。  
- **目前唯一觀測到的 Issue**：`#1` 只包含標題，缺乏任何描述、指派、標籤或討論。
