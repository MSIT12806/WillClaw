# Repository Memory

## Stable Context
- **Repository**: `MSIT12806/WillClaw`  
- **Issue Management**  
  - 所有工作皆以 GitHub Issue 為單位追蹤。  
  - 每個 Issue 必須在建立時提供 **標題**、**具體描述**、**目標與預期成果**，否則會被龍蝦標記為 *資訊不明*，無法分配任務或產生決策。  
  - 只有在 Issue 內容完整、得到作者回覆後，龍蝦才會開始 **任務分解、標籤分配、負責人指派** 等後續流程。  
- **龍蝦團隊行為準則**（根據 `shared/manual.md`）  
  1. **不直接複製原始 Issue 文字**，僅抽取關鍵資訊作為長期記憶。  
  2. **穩定規則**、**長期決策**、**常見限制** 必須寫入此檔案，供所有 agent 共享。  
  3. **Compact‑memory workflow** 會讀取本檔案，但不會覆寫，確保手動維護的內容永續保存。  
- **工作節奏**  
  - 龍蝦每日檢查過去 30 天內的開放 Issue，彙整「跨議題主題」與「未解迴路」。  
  - 若同一天內出現多次相同主題（如需求缺口），會提升為 **Stable Context** 或 **Recent Themes**，視其持續性而定。
